Digital Wallet Database System

This repository contains the complete database project for a Digital Wallet System. The project demonstrates database design, implementation, and query management using PostgreSQL, including the creation of tables, relationships, queries, and normalization.

Project Overview
The Digital Wallet System allows users to manage their funds digitally, make payments, add money, use offers, and manage beneficiaries. Administrators oversee merchants, users, and transactions, while KYC documents ensure regulatory compliance.

Key features include:
-User account management
-Wallet management
-Payment methods integration
-Merchant and offer management
-Transaction tracking
-Beneficiary management
-KYC document verification

Project Contents

The project includes
ER Diagram
Visual representation of entities, attributes, and relationships.
Shows primary keys (PK), foreign keys (FK), and cardinality between entities.
Relational Schema
Transforms the ER diagram into relational tables.
Defines table structures, data types, and constraints.
SQL Queries
DDL queries to create tables: admin, users, wallet, merchant, offer, payment_method, transaction, beneficiary, kyc_document.
DML queries with sample data (10 records per table).
Select queries demonstrating joins, aggregates, and conditional filters.
Normalization
Includes 1NF, 2NF, and 3NF tables.
Ensures elimination of redundancy and dependency anomalies.

Project Report
Includes project scope, ER diagram, relational schema, normalization details.
